Overview

MicroOS is a minimal Linux distribution built for efficiency and speed. It consists of:

  • Linux kernel 6.18.5-arch1-1
  • GRUB bootloader
  • Custom init system written in C
  • Custom terminal shell written in Rust
  • Essential system utilities written in C/C++

The entire system is designed to run in memory-constrained environments while providing basic functionality for system administration and file management.

System Architecture

MicroOS follows a monolithic kernel architecture with a minimal userland:

Applications

User utilities: file managers, system tools

Terminal Shell

Rust-based interactive shell with command execution

Init System

Custom init daemon managing process lifecycle

Linux Kernel

Linux 6.18.5 with essential drivers and subsystems

Hardware

x86_64 compatible processors and peripherals

Boot Process

  1. BIOS/UEFI loads GRUB bootloader
  2. GRUB loads Linux kernel and initramfs
  3. Kernel initializes hardware and mounts root filesystem
  4. Init process (PID 1) starts:
    • Mounts /proc, /sys, /dev filesystems
    • Sets up console
    • Launches the terminal shell
  5. Terminal provides interactive command interface

Total boot time is typically under 10 seconds on most hardware.

File System Structure

/
├── bin/           # Essential user binaries
├── sbin/          # System binaries (init)
├── etc/           # Configuration files
│   ├── help.txt
│   └── microos-ver.txt
├── dev/           # Device files
├── proc/          # Process information
├── sys/           # Kernel objects
├── tmp/           # Temporary files
│   ├── buffer
│   └── copy_filename.txt
└── usr/
    ├── bin/       # Additional binaries
    └── sbin/      # Additional system binaries

The root filesystem is loaded into memory (initramfs), providing fast access but volatile storage. For persistence, use the optional data partition.

Terminal Usage

The MicroOS terminal provides a simple command-line interface with:

  • Command history (with arrow keys if TTY supported)
  • Tab completion (limited)
  • Signal handling (Ctrl+C to interrupt)
  • Built-in commands: clear, exit, pcd, gtd, help

The terminal searches for executables in /bin, /sbin, /usr/bin, /usr/sbin directories.

Networking

MicroOS currently does not include networking capabilities in the base system. This is a design choice to maintain minimal size. Network functionality can be added by:

  1. Compiling kernel with network drivers
  2. Adding network utilities (ifconfig, ping, etc.)
  3. Implementing DHCP client

See the GitHub repository for instructions on adding networking support.

Troubleshooting

System won't boot

Solution: Verify USB creation process. Use dd with correct device name. Check BIOS boot order.

No TTY support (arrow keys don't work)

Solution: Kernel lacks proper TTY drivers for your hardware. Basic input still works.

Command not found

Solution: Check command spelling. Use sd /bin to see available commands.

Permission denied

Solution: Use chperm to modify permissions if needed.

Out of memory

Solution: Ensure system has at least 125MB RAM. Close unnecessary processes with stop command.
